# Git 和 GitHub 教程（Ubuntu）

## 课程目标

完成本教程后，你将能够：

- 在 Ubuntu 上安装 Git
- 在 GitHub 上创建仓库
- 将远程仓库克隆到本地
- 将本地代码提交并推送到 GitHub
- 创建并切换分支
- 安全地合并分支

---

## 1. 在 Ubuntu 上安装 Git

### 1.1 更新软件源并安装 Git

```bash
sudo apt update
sudo apt install -y git
```

### 1.2 验证安装结果

```bash
git --version
```

你应当看到类似输出：

```text
git version 2.xx.x
```

### 1.3 配置 Git 身份信息

设置全局用户名与邮箱（会写入提交记录）：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

检查配置：

```bash
git config --global --list
```

---

## 2. 将 Ubuntu 连接到 GitHub

你可以使用 HTTPS（搭配 Token）或 SSH。长期使用推荐 SSH。

### 2.1 生成 SSH 密钥

```bash
ssh-keygen -t ed25519 -C "你的邮箱@example.com"
```

**不要设置口令！！！不要设置口令！！！**

### 2.2 启动 ssh-agent 并添加私钥

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

**如果你设置了密码，那就用这个命令。**

### 2.3 复制公钥

```bash
cat ~/.ssh/id_ed25519.pub
```

复制输出内容后，进入 GitHub：

- Settings
- SSH and GPG keys
- New SSH key
- 粘贴并保存

### 2.4 测试连接

```bash
ssh -T git@github.com
```

连接成功时，GitHub 会显示你的用户名欢迎信息。

## 3. 在 GitHub 上创建仓库

### 3.1 通过 GitHub 网页创建

1. 点击右上角 +
2. 选择 New repository
3. 填写仓库名（描述可选）
4. 选择 Public 或 Private
5. 点击 Create repository

创建完成后，GitHub 会显示将本地代码推送到该仓库的示例命令。

### 3.2 （可选）把已有本地项目关联到新仓库

在你的本地项目目录执行：

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin git@github.com:<your-username>/<your-repo>.git
git push -u origin main
```

---

## 4. 克隆仓库

### 4.1 使用 SSH 克隆

```bash
git clone git@github.com:<owner>/<repo>.git # 推荐
```

### 4.2 使用 HTTPS 克隆

```bash
git clone https://github.com/<owner>/<repo>.git
```

### 4.3 进入项目目录

```bash
cd <repo>
```

---

## 5. 上传代码：Add、Commit、Push

这是日常最常用的工作流。

### 5.1 查看当前状态

```bash
git status
```

### 5.2 添加到暂存区

```bash
git add .
```

也可以只添加单个文件：

```bash
git add src/main.py
```

### 5.3 提交更改

```bash
git commit -m "Add login page"
```

### 5.4 推送到 GitHub

```bash
git push
```

如果是新分支第一次推送：

```bash
git push -u origin <branch-name> # 远程的名字
```

---

## 6. 创建和使用分支

分支可以让你在不影响 main 的情况下开发新功能。这个会经常使用，所以学会如何开分支和合并分支是很重要的一个技能。

### 6.1 查看分支

```bash
git branch
```

### 6.2 创建新分支

```bash
git branch feature/login
```

### 6.3 切换到分支

```bash
git checkout feature/login
```

### 6.4 一步创建并切换

```bash
git checkout -b feature/login
```

较新的等价写法：

```bash
git switch -c feature/login
```

### 6.5 将分支推送到 GitHub

```bash
git push -u origin feature/login
```

---

## 7. 合并分支

### 7.1 在本地合并

假设你已完成 feature/login，需要把它合并到 main：

```bash
git checkout main
git pull
git merge feature/login
git push
```

### 7.2 使用 Pull Request 合并（团队协作推荐）

1. 将功能分支推送到 GitHub
2. 打开仓库页面
3. 点击 Compare & pull request
4. 代码评审并创建 PR
5. 将 PR 合并到 main

简单例子，比如说我写这一次课件，就是先在仓库中点击一次fork，这个操作会把main分支推送到我的仓库我完成我的部分之后，点击compare & pull request，然后点击create pull request，然后蔡老师审查代码之后点击merge，然后点击confirm merge，然后点击close pull request。
这样，就完成了一次代码的贡献

## 8. 冲突处理（重要）

当两个分支修改了同一位置代码时，可能出现合并冲突。

典型处理流程：

```bash
git merge feature/login
# 在文件中手动解决冲突
git add .
git commit -m "Resolve merge conflict"
git push
```

冲突时，文件中常见标记如下：

```text
<<<<<<< HEAD
当前分支内容
=======
待合并分支内容
>>>>>>> feature/login
```

保留最终正确内容后，删除这些标记即可。

## 9. 常用日常命令

```bash
git status                # 查看仓库状态
git log --oneline --graph --decorate --all
git remote -v             # 查看远程仓库地址
git pull                  # 拉取并合并远程更新
git fetch                 # 仅拉取远程更新
git diff                  # 查看未暂存改动
git diff --staged         # 查看已暂存改动
```

## 10. 建议练习流程

完整走一遍下面流程：

1. 在 GitHub 创建一个仓库
2. 在 Ubuntu 上克隆该仓库
3. 新建一个文件并提交
4. 推送到 main
5. 创建 feature/test 分支
6. 在分支修改文件并提交
7. 推送分支并发起 Pull Request
8. 将分支合并到 main
9. 本地拉取最新 main

如果你完整走通这套流程，就已经掌握了 GitHub 的核心协作流程。

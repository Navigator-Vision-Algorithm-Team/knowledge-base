### 2.1 在 Linux 上安装 VSCode

- 在浏览器找到vsc的官方网站，下载Linux版本安装包。

- 自行研究如何在Linux中使用安装包安装软件

### 2.2 配置 SSH 连接虚拟机

在虚拟机里查自己的 IP：

```bash
ip addr show | grep "inet "
```

找到类似 `192.168.x.x` 的地址（不是 127.0.0.1）。

在 Windows 的 VSCode 里：
1. 按 `Ctrl+Shift+P`，输入 `Remote-SSH: Connect to Host`
2. 输入 `用户名@虚拟机IP`，例如 `alice@192.168.56.101`
3. 按提示输入密码

连上之后，VSCode 的左下角会显示远程主机信息，说明成功了。

> 如果 SSH 连不上，去虚拟机里执行 `sudo apt install openssh-server` 再试。

## 模块三：检查 ROS 环境并学习基础

**目标：** 确认 ROS 安装正确，能跑通最基本的节点通信示例。

### 3.1 确认 ROS 版本

我们使用 **ROS 2 Humble**（对应 Ubuntu 22.04）。

```bash
# 检查 ROS 2 是否安装
ros2 --version

# 查看 ROS 2 环境变量是否生效
echo $ROS_DISTRO
```

如果 `ros2` 命令找不到，说明环境变量没有 source。执行：

```bash
source /opt/ros/humble/setup.bash

# 把它加进 ~/.bashrc，以后每次开终端自动生效
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

如果 ROS 2 根本没安装，参考鱼香ros

### 3.2 跑通 talker / listener 示例

这是 ROS 2 的 Hello World，必须跑通。

打开**两个**终端窗口（VSCode 里开两个终端也行）：

**终端 1（发布者）：**
```bash
ros2 run demo_nodes_cpp talker
```

**终端 2（订阅者）：**
```bash
ros2 run demo_nodes_cpp listener
```

你应该能看到 talker 不断发送消息，listener 不断接收。两个终端都用 `Ctrl+C` 停止。

### 3.3 学习 ROS 2 基本概念

自己查资料（ROS 2 官方文档或问 AI），搞清楚以下问题：

1. **Node（节点）** 是什么？
2. **Topic（话题）** 是什么？Publisher 和 Subscriber 怎么用？
3. **Service（服务）** 和 Topic 有什么区别？

用自己的话写下来，不超过三句话。不需要完全准确，有自己的理解就行。

### 3.4 查看话题信息

在 talker 运行时，在第三个终端执行：

```bash
# 查看当前所有活跃话题
ros2 topic list

# 查看 /chatter 话题的消息
ros2 topic echo /chatter

# 查看话题发布频率
ros2 topic hz /chatter

# 查看节点列表
ros2 node list
```

### 3.5 检查点

- [ ] `ros2 --version` 有输出

- [ ] 能用 `ros2 topic list` 和 `ros2 topic echo` 查看话题

## 模块四：搞清楚 VPN 对虚拟机网络的影响

**目标：** 明白开了 VPN 之后，Windows 宿主机和 Ubuntu 虚拟机的网络是各自独立的，互不影响。

### 4.1 先理解网络结构

虚拟机和宿主机之间有一个**虚拟网卡**（VMnet）单独通信，和宿主机的物理网卡是分开的。

常用的虚拟机网络模式：

| 模式 | 说明 |
|------|------|
| NAT | 虚拟机通过宿主机上网，宿主机能 ping 到虚拟机（推荐） |
| 桥接 | 虚拟机和宿主机在同一局域网，IP 由路由器分配 |
| 仅主机 | 虚拟机只能和宿主机通信，不能访问外网 |

**推荐用 NAT 模式。**

### 4.2 验证：开 VPN 前后，SSH 连接不受影响

**实验步骤：**

1. 关闭 VPN，确认 VSCode Remote SSH 能连上虚拟机
2. 记下虚拟机的 IP（`ip addr`）
3. 开启 VPN
4. 再次用同一个 IP 通过 VSCode Remote SSH 连接虚拟机

**正常情况下，VPN 开关不影响你 SSH 连虚拟机**，因为走的是 VMnet 虚拟网卡，不经过 VPN 隧道。

如果 VPN 开了之后 SSH 断了，说明你的 VPN 开了全局代理，把 VMnet 流量也劫持了。解决方法：

```
在 VPN 客户端设置里，把 192.168.x.x/24（虚拟机网段）加入"直连"或"绕过代理"列表。
```

### 4.3 验证：虚拟机访问外网不依赖宿主机 VPN

1. 在宿主机开启 VPN
2. 在虚拟机里执行：
   ```bash
   curl -I https://www.google.com
   ```
3. 如果请求超时，说明虚拟机不走宿主机的 VPN（这是正常的）
4. 虚拟机要用代理，需要在虚拟机里单独配置

### 4.4 虚拟机里配置代理（可选）

如果需要让虚拟机访问被墙的网站（比如下 ROS 包），在虚拟机里配置：

```bash
# 临时设置（关终端后失效）
export http_proxy=http://宿主机IP:代理端口
export https_proxy=http://宿主机IP:代理端口

# 测试
curl -I https://www.google.com

# 永久写入（不推荐，容易忘了导致奇怪问题）
echo 'export http_proxy=http://宿主机IP:端口' >> ~/.bashrc
```

宿主机 IP 就是 VMnet 网卡的地址（通常是 `192.168.x.1`），代理端口看你的 VPN 客户端设置。

### 4.5 检查点

- [ ] 理解 NAT / 桥接 / 仅主机三种网络模式的区别
- [ ] 验证过开 VPN 前后 SSH 连虚拟机是否受影响
- [ ] 知道虚拟机的网络和宿主机 VPN 是独立的
- [ ] （可选）能在虚拟机里配置代理访问外网

### 5.1 文件权限练习

```bash
# 创建一个脚本
mkdir -p ~/perm_test
cat > ~/perm_test/hello.sh << 'EOF'
#!/bin/bash
echo "Hello, I am a script!"
echo "Current user: $(whoami)"
echo "Current dir: $(pwd)"
EOF

# 查看当前权限
ls -l ~/perm_test/hello.sh

# 尝试直接运行（会失败）
~/perm_test/hello.sh

# 加执行权限
chmod +x ~/perm_test/hello.sh

# 再次运行（成功）
~/perm_test/hello.sh

# 用数字方式修改权限
chmod 644 ~/perm_test/hello.sh   # 去掉执行权限
chmod 755 ~/perm_test/hello.sh   # 恢复
```

**问题：** `644` 和 `755` 分别代表什么权限？自己算一下。

### 5.3 GCC 编译练习

**任务 1：编译并运行一个带命令行参数的 C 程序**

用 Vim 创建 `~/gcc_test/args.c`：

```c
#include <stdio.h>
#include <stdlib.h>

int main(int argc, char *argv[]) {
    if (argc < 3) {
        printf("用法: %s <数字1> <数字2>\n", argv[0]);
        return 1;
    }
    int a = atoi(argv[1]);
    int b = atoi(argv[2]);
    printf("%d + %d = %d\n", a, b, a + b);
    printf("%d * %d = %d\n", a, b, a * b);
    return 0;
}
```

```bash
mkdir -p ~/gcc_test
# 用 vim 创建上面的文件，然后：
gcc -Wall ~/gcc_test/args.c -o ~/gcc_test/args
~/gcc_test/args 6 7
```
